## 📂 STEP 2: Project Setup (10 minutes)

### 1. Create Project Structure
```bash
cd /Users/user/Desktop/RPSS/eventra
mkdir -p src/main/java/com/eventra/eventra/{model,repository,service,controller,enums}
mkdir -p src/main/resources/templates/{auth,event,dashboard,admin,feedback}
mkdir -p src/main/resources/static/{css,js,images}
mkdir -p src/test/java/com/eventra/eventra/{service,controller}
mkdir -p src/test/resources
```

### 2. Update pom.xml
- Copy the dependencies from `IMPLEMENTATION_GUIDE.md`
- Add Lombok, Spring Security, Validation, Testing libraries

### 3. Create application.properties
```properties
# File: src/main/resources/application.properties

spring.datasource.url=jdbc:mysql://localhost:3306/rpss_db
spring.datasource.username=root
spring.datasource.password=your_password
spring.jpa.hibernate.ddl-auto=update
spring.thymeleaf.prefix=classpath:/templates/
```

### 4. Create application-test.properties
```properties
# File: src/test/resources/application-test.properties

spring.datasource.url=jdbc:h2:mem:testdb
spring.datasource.username=sa
spring.jpa.hibernate.ddl-auto=create-drop
```

## 📋 Clarification: Your Requirements vs Implementation

### ✅ Admin Roles (Confirmed)
- **Admin = HOD Sir + Activity Coordinator + Office Bearers**
- HOD: Head of Department (academic)
- Activity Coordinator: Academic staff managing activities
- Office Bearers: President, Vice President, Secretary, Treasurer, Student Oracle (elected students)
- **Implementation**: All have Role = ADMIN (same permissions)
- Can approve/reject events, manage clubs & users, generate reports

### ✅ Clubs (7 Predefined + Future Expansion)
- Codeverse Technical Club
- Kalakruti Cultural Club
- Strikers Sports Club
- Innovation Club
- Digisphere Digital marketing Club
- Samvedna Social Club
- Photography Club

**Future Addition**: Admin → Manage Clubs → Add New Club

### ✅ Payment (Optional Per Event)
- Free events: `requiresPayment = false`, `paymentAmount = NULL`
- Paid events: `requiresPayment = true`, `paymentAmount = 500` (example)
- Flow: Registration → Check if paid → If yes, redirect to payment gateway
- Payment Status: NOT_REQUIRED → PENDING → COMPLETED

### ✅ User ID (KEEP)
- `userId` is JPA Primary Key (auto-generated by database)
- NOT redundant - it's the database identifier
- Different from role-based identification

### ✅ NO REST API
- Using Thymeleaf MVC model
- Form submissions → Controllers → Templates
- Session-based authentication (not JWT)
- Testing with MockMvc + HTML assertions

## 🎓 Key Concepts for Viva Preparation

### 1. Event Lifecycle
```
PENDING → APPROVED → COMPLETED → REPORT
  ↓         ↓
REJECTED  CANCELLED
```

### 2. Registration Flow
```
Free Event:
  Browse → Register → Download Pass → Attend → Feedback

Paid Event:
  Browse → Register → Payment → Pass → Attend → Feedback
                        ↓
                   If failed: Retry or Cancel
```

### 3. Role-Based Access
```
PARTICIPANT:
✓ View approved events
✓ Register for events
✓ Make payment (if required)
✓ Download pass
✓ Give feedback
✗ Approve events
✗ Create events

CLUB_HEAD:
✓ Create events
✓ Submit event for approval
✓ View participant list
✓ Upload media/photos
✓ Generate event report
✗ Approve events (only ADMIN)
✗ Manage other clubs

ADMIN:
✓ Approve/Reject events
✓ Manage all users
✓ Manage all clubs
✓ Add new clubs
✓ View analytics
✓ Generate system reports
✓ Manage office bearers
```

## 🛠️ Common Implementation Tasks

### Task 1: Create a Participant User
```java
User participant = User.builder()
    .name("John Student")
    .email("john@example.com")
    .phone("9876543210")
    .role(participantRole)
    .build();
participant.encryptPassword("securePassword123");
userRepository.save(participant);
```

### Task 2: Create Free Event
```java
Event freeEvent = Event.builder()
    .title("RPSS Orientation")
    .description("Orientation for all students")
    .eventDate(LocalDateTime.now().plusDays(7))
    .venue("Auditorium")
    .maxCapacity(500)
    .club(club)
    .createdBy(clubHead)
    .requiresPayment(false)  // ← Free event
    .build();
eventRepository.save(freeEvent);
```

### Task 3: Create Paid Event
```java
Event paidEvent = Event.builder()
    .title("Premium Workshop")
    .description("Advanced web development")
    .eventDate(LocalDateTime.now().plusDays(14))
    .venue("Lab")
    .maxCapacity(30)
    .club(club)
    .createdBy(clubHead)
    .requiresPayment(true)           // ← Paid
    .paymentAmount(new BigDecimal("500"))  // ← Amount
    .build();
eventRepository.save(paidEvent);
```

### Task 4: Register for Free Event
```java
// Check if space available
if (freeEvent.isSpaceAvailable()) {
    Registration registration = Registration.builder()
        .event(freeEvent)
        .participant(participant)
        .status(RegistrationStatus.CONFIRMED)
        .paymentStatus(PaymentStatus.NOT_REQUIRED)
        .build();
    registrationRepository.save(registration);
}
```

### Task 5: Approve Event (Admin)
```java
Event event = eventRepository.findById(eventId).get();
event.approveEvent(admin);  // Sets status=APPROVED, approvedBy, approvalDate
eventRepository.save(event);
```

### Task 6: Generate QR Pass
```java
Registration registration = registrationRepository.findById(regId).get();
String qrCodeData = generateQRCode(registration);
Pass pass = Pass.builder()
    .registration(registration)
    .qrCode(qrCodeData)
    .qrImagePath("/passes/" + registration.getId() + ".png")
    .build();
passRepository.save(pass);
```

### Task 7: Submit Feedback
```java
Feedback feedback = Feedback.builder()
    .event(event)
    .participant(participant)
    .rating(5)  // 1-5 stars
    .comment("Excellent event!")
    .build();
feedbackRepository.save(feedback);
```

### Task 8: Generate Event Report
```java
Double avgRating = feedbackRepository.getAverageRatingByEvent(eventId);
Long participantCount = registrationRepository.countByEventEventId(eventId);
BigDecimal totalRevenue = calculateTotalRevenue(eventId);

Report report = Report.builder()
    .event(event)
    .totalParticipants(participantCount.intValue())
    .totalRevenue(totalRevenue)
    .averageRating(BigDecimal.valueOf(avgRating))
    .feedbackCount((int)feedbackRepository.countByEventEventId(eventId))
    .build();
reportRepository.save(report);
```

## 🧪 Example: Testing Event Approval Workflow

```java
@Test
void testEventApprovalWorkflow() {
    // Setup
    Event event = createEvent("Test Event", EventStatus.PENDING);
    User admin = createAdminUser();

    // Club head creates event
    assertEquals(EventStatus.PENDING, event.getStatus());

    // Admin approves
    event.approveEvent(admin);
    eventRepository.save(event);

    // Verify
    Event updated = eventRepository.findById(event.getEventId()).get();
    assertEquals(EventStatus.APPROVED, updated.getStatus());
    assertEquals(admin, updated.getApprovedBy());
    assertNotNull(updated.getApprovalDate());
}
```

## ❓ FAQ

**Q: Where to put properties file?**
A: `src/main/resources/application.properties`

**Q: What if database password differs?**
A: Update `application.properties` with your MySQL password

**Q: How to test without full frontend?**
A: Use Integration tests with MockMvc (Postman not needed)

**Q: Can I add more clubs later?**
A: Yes! Admin dashboard → Manage Clubs → Add New Club

**Q: Payment gateway integration?**
A: Out of scope for MVP. For now:
